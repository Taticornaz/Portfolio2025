Réflexion finale sur mon parcours de formation

Introduction

La réalisation de ce portfolio représente l’aboutissement de mon parcours d’apprentissage en développement web front-end. Ce projet m’a permis de mettre en pratique les compétences techniques acquises pendant la formation et de réfléchir à ma progression, aux difficultés rencontrées et aux compétences que je souhaite continuer à développer.

Ce portfolio regroupe plusieurs projets web, des fiches techniques sur les technologies apprises ainsi qu’une réflexion critique sur mon apprentissage.

Projets et apprentissages

Gym Website

Qu’ai-je appris ? Structurer une landing page responsive avec HTML, CSS et JavaScript ; intégrer un formulaire de contact et des animations simples.

Difficultés et causes : Mise en page responsive difficile sur mobile ; certains effets JS nécessitaient des ajustements précis.

Actions pour la suite : Tester les breakpoints dès le début, documenter les étapes de CSS et JS pour maintenance future.

Motivational Quotes

Qu’ai-je appris ? Génération de contenu dynamique avec JavaScript et design responsive.

Difficultés et causes : Comprendre la manipulation du DOM pour mettre à jour les citations ; gestion de l’aléatoire.

Actions pour la suite : Découper le code en fonctions modulaires et tester chaque fonctionnalité individuellement.

Personal Profile

Qu’ai-je appris ? Création d’une page profil claire et esthétique avec Tailwind CSS et structure HTML sémantique.

Difficultés et causes : Utiliser Tailwind CSS correctement et adapter les sections pour le responsive design.

Actions pour la suite : Planifier la structure responsive dès la conception et tester sur différents écrans.

Quiz App

Qu’ai-je appris ? Développer un quiz interactif avec feedback immédiat et score final en JavaScript.

Difficultés et causes : Gestion des événements et de l’état du quiz complexe pour un débutant JS.

Actions pour la suite : Utiliser un diagramme de flux pour planifier les interactions et organiser le code en fonctions réutilisables.

Weather App

Qu’ai-je appris ? Récupérer et afficher des données externes via API et créer une interface responsive.

Difficultés et causes : Compréhension des requêtes fetch et gestion des erreurs API.

Actions pour la suite : Toujours vérifier les réponses API et ajouter des messages d’erreur clairs pour l’utilisateur.

Learn Words App (React)

Qu’ai-je appris ? Création d’une application avec React, composants réutilisables, gestion de l’état et cartes interactives.

Difficultés et causes : Conflits de version npm pour React ; compréhension de l’état et des props entre composants.

Actions pour la suite : Commencer avec architecture modulaire claire, documenter les étapes, vérifier les dépendances npm avant de lancer l’installation.

Compétences techniques acquises

Au cours de cette formation, j’ai acquis une base solide dans plusieurs technologies fondamentales du développement web.

  • Structuration de pages web avec HTML5 et balises sémantiques
  • Mise en page avec CSS3 et Flexbox
  • Création d’interfaces responsive adaptées aux différents écrans
  • Développement d’interactions avec JavaScript
  • Utilisation d’API pour récupérer des données dynamiques
  • Gestion de projet et versioning avec Git et GitHub

Ces compétences m’ont permis de réaliser plusieurs projets concrets et de comprendre les bases du développement front-end moderne.

Projet réalisé avec React

Dans le cadre de ma formation, j’ai également développé une application de cartes mémoire (flashcards) pour apprendre des mots en utilisant React.

Cette application permet d’afficher des cartes interactives et représente mon premier projet utilisant un framework JavaScript moderne.

Grâce à ce projet, j’ai découvert plusieurs concepts importants :

  • La création de composants React
  • La gestion de l’état (state)
  • L’organisation d’une application en composants réutilisables
  • La logique d’interactions dans une application dynamique

Difficultés techniques rencontrées avec React

Lors du développement de cette application, j’ai rencontré un problème technique lié à l’installation des dépendances avec npm.

Certaines bibliothèques utilisaient des versions différentes de React, ce qui provoquait des conflits de dépendances lors de l’installation.

Pour résoudre ce problème, j’ai dû analyser les messages d’erreur, vérifier les versions des packages et ajuster l’installation des dépendances.

Cette expérience m’a permis de mieux comprendre :

  • le fonctionnement de npm et des dépendances
  • l’importance de la compatibilité entre les versions des librairies
  • la résolution de problèmes techniques dans un projet réel

Même si cette difficulté a ralenti le développement du projet, elle a été très formatrice et m’a permis de progresser dans ma compréhension de l’écosystème React.

Difficultés rencontrées pendant la formation

Au cours de mon apprentissage, plusieurs défis sont apparus.

  • Comprendre la logique du responsive design
  • Organiser correctement la structure du code HTML et CSS
  • Mettre en place certaines fonctionnalités JavaScript
  • Structurer un projet comportant plusieurs sections et fichiers
  • Comprendre les bases du fonctionnement de React

Ces difficultés ont été importantes pour développer une approche plus méthodique du développement.

Comment j’ai dépassé ces difficultés

  • Analyse des problèmes et recherche de solutions
  • Tests progressifs pour comprendre le comportement du code
  • Réorganisation du code pour améliorer sa lisibilité
  • Consultation de la documentation technique
  • Pratique régulière à travers différents projets

Grâce à cette approche, j’ai progressivement amélioré mes compétences et ma capacité à résoudre des problèmes techniques.Chaque difficulté m’a obligée à réfléchir non seulement à la solution immédiate, mais aussi à la stratégie de travail et à l’organisation du projet. Par exemple, les conflits de versions npm m’ont appris l’importance de vérifier la compatibilité des dépendances avant de commencer un projet. Les tests sur différents écrans et navigateurs m’ont sensibilisée à l’anticipation et à la planification pour un design vraiment responsive. Cette approche me permettra de mieux gérer les futurs projets et de construire un portfolio professionnel de qualité.

Importance des fiches techniques

La création de fiches techniques a joué un rôle important dans mon apprentissage.

Elles m’ont permis de structurer mes connaissances et de mieux comprendre les concepts clés du développement web comme l’accessibilité, le responsive design, JavaScript, les API ou encore React.

Ces fiches constituent également une base de connaissances que je pourrai réutiliser dans mes futurs projets.

Perspectives d’évolution

Même si cette formation m’a permis d’acquérir de bonnes bases, le développement web est un domaine en constante évolution.

Je souhaite continuer à progresser dans plusieurs domaines :

  • approfondir mes compétences en React
  • améliorer ma maîtrise de JavaScript
  • développer des applications web plus complexes
  • renforcer mes connaissances en accessibilité web
  • découvrir d’autres outils du développement front-end

Amélioration continue

Cette formation m’a également appris que le développement web est un processus d’apprentissage continu.

Chaque projet apporte de nouvelles connaissances et de nouveaux défis. Je souhaite continuer à pratiquer régulièrement, à explorer de nouvelles technologies et à améliorer la qualité de mon code et de mes projets.

Bilan final

La réalisation de ce portfolio constitue une étape importante dans mon parcours vers le développement web.

Ce projet m’a permis de comprendre qu’un bon site web ne dépend pas uniquement de son apparence visuelle, mais aussi de la qualité de sa structure, de son code et de l’expérience utilisateur.

Aujourd’hui, je suis capable de concevoir des sites web structurés, responsive et interactifs, ainsi que de créer une première application avec React.

Ce portfolio représente donc une base solide pour la suite de mon évolution en tant que développeuse web.
