Réflexion critique sur la création de mon portfolio web

Qu’ai-je appris ?

La réalisation de mon portfolio m’a permis de consolider mes bases en développement front-end tout en comprenant que la création d’un site web ne se limite pas à l’assemblage de sections visuelles. J’ai appris à structurer une page complète avec un seul fichier HTML, à organiser le contenu de manière logique et à utiliser des techniques modernes pour obtenir une mise en page responsive et maintenable.

Plus précisément, ce projet m’a permis d’apprendre :

  • L’utilisation de Flexbox pour aligner et organiser les sections (navigation, projets, About me) de façon flexible
  • L’importance des media queries pour adapter le design aux écrans mobiles, tablettes et desktop
  • La structuration d’un portfolio en sections claires et navigables via une sidebar
  • L’intégration de JavaScript pour améliorer l’expérience utilisateur, notamment avec un effet de texte animé
  • L’intégration d’un formulaire de contact fonctionnel via Formspree

Ce projet m’a fait comprendre que chaque choix technique a un impact direct sur la lisibilité du code, la maintenance et l’expérience utilisateur.

Quelles difficultés ai-je rencontrées ?

  • Gérer un projet relativement conséquent dans un seul fichier HTML
  • Responsive design : certaines sections fonctionnaient sur desktop mais pas sur mobile (alignement, espacements)
  • JavaScript : mise en place de l’effet de texte animé difficile à comprendre au début
  • Formulaire de contact : réflexion sur l’UX, clarté des champs et simplicité de l’interaction

Comment ai-je dépassé ces difficultés ?

  • Réorganisation du code HTML avec des balises sémantiques (section, nav, main, aside)
  • Approche méthodique du responsive design : breakpoints cohérents + media queries
  • Utilisation de Flexbox pour éviter le positionnement absolu rigide
  • Décomposition de la logique JavaScript de l’animation en étapes simples (écriture, pause, suppression)
  • Utilisation de Formspree pour un formulaire fiable sans backend

Compétences à acquérir ou consolider

  • Accessibilité web : labels, contrastes, navigation clavier
  • JavaScript avancé : fonctions modulaires et structure du code
  • Optimisation responsive : approche mobile-first
  • Organisation de projet : séparation HTML, CSS, JavaScript dans projets plus complexes
  • Frameworks modernes : approfondir React

Valeur ajoutée de cet apprentissage

La création de ce portfolio a profondément changé ma manière d’aborder le développement web. Je suis passée d’une logique « faire fonctionner le site » à une logique « construire un projet propre, maintenable et évolutif ».

J’ai compris que le développement front-end ne concerne pas uniquement le design, mais aussi : la structure, la lisibilité du code, l’expérience utilisateur et l’adaptabilité aux différents supports.

Cette expérience constitue une base solide pour mes futurs projets et marque une étape importante dans ma progression vers une pratique plus professionnelle du développement web.
