Fetch API

Définition concise

Fetch permet de récupérer des données depuis un serveur sans rechargement de page. C'est l'API moderne pour les requêtes HTTP en JavaScript.

Contexte d'utilisation

  • Récupérer des données JSON d'une API
  • Envoyer des données au serveur (POST, PUT, DELETE)
  • Gestion asynchrone avec async/await ou .then()
  • Gérer les erreurs et les réponses

Cas d'usage réel

Dans Portfolio2025, j'ai utilisé Fetch pour récupérer des données de projets et les afficher dynamiquement sans rechargement de page.

Exemple de code


// Avec async/await
async function getProjects() {
  try {
    const response = await fetch('https://api.example.com/projects');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Erreur:', error);
  }
}

// Avec .then()
fetch('https://api.example.com/projects')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Pièges à éviter

  • Oublier d'appeler .json() sur la réponse
  • Ne pas gérer les erreurs avec try/catch ou .catch()
  • Utiliser fetch sans vérifier le statut de la réponse
  • Faire des appels API trop fréquents sans throttling

Analyse personnelle

Fetch a vraiment simplifié la récupération de données dans mes projets. C'est beaucoup plus lisible que XMLHttpRequest, et async/await rend le code très clair.
