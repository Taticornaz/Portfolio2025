Déploiement et mise en production

Définition concise

Le déploiement consiste à publier le site sur un serveur accessible au public, et à configurer la production pour performance, sécurité et fiabilité.

Contexte d'utilisation

  • Hébergement sur GitHub Pages, Netlify ou Vercel
  • Configuration des builds, minification CSS/JS
  • Optimisation des assets pour réduire le temps de chargement
  • Tests finaux avant publication

Cas d'usage réel

Pour Portfolio2025, j’ai déployé le site sur GitHub Pages, optimisé les images et le CSS, et vérifié le responsive et les performances avant de partager le lien public.

Pièges à éviter

  • Publier sans tests finaux
  • Oublier d’optimiser les images ou le CSS
  • Ne pas configurer correctement HTTPS ou le nom de domaine

Analyse personnelle

J’ai compris que le déploiement ne se limite pas à « mettre en ligne ». Il faut tester, optimiser et sécuriser le site pour offrir une expérience fiable à tous les utilisateurs.
