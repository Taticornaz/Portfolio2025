React – Gestion de mots avec des composants

Définition concise

Cette fiche présente comment gérer un état dynamique d'une liste de mots dans React à l'aide de composants, de hooks et de la manipulation d'événements. L'exemple illustre l'édition en direct de mots et de leurs traductions.

Contexte d'utilisation

Cette technique est utile pour :

  • Créer des applications interactives avec des listes de données
  • Gérer l'état local avec useState
  • Mettre à jour dynamiquement les composants enfants via des props et callbacks

Exemple de code commenté


import React, { useState } from 'react';
import WordCard from '../WordCard/WordCard';

const MainPage = () => {
  const [words, setWords] = useState([
    { id: 1, word: 'mot1', translation: 'traduction1' },
    { id: 2, word: 'mot2', translation: 'traduction2' },
  ]);

  const handleEdit = (id, updatedWord, updatedTranslation) => {
    setWords(prevWords =>
      prevWords.map(word =>
        word.id === id ? { ...word, word: updatedWord, translation: updatedTranslation } : word
      )
    );
  };

  return (
    <div>
      <h1>Table des mots</h1>
      {words.map(word => (
        <WordCard key={word.id} word={word} onEdit={handleEdit} />
      ))}
    </div>
  );
};

export default MainPage;

Cas d'usage réel

Dans mon projet LearnWords, j'utilise ce composant pour :

  • Afficher une liste de mots avec leur traduction
  • Permettre à l'utilisateur de modifier directement les mots existants
  • Maintenir l'état de la liste dans le composant parent et passer des callbacks aux enfants

Analyse personnelle

En développant ce projet, j'ai rencontré un problème concret : j'avais initialement installé React 19, mais certaines dépendances comme @testing-library/react nécessitaient React 18. Cela a causé des erreurs lors du déploiement sur Netlify (peer dependency conflict). J'ai dû downgrader React et React-DOM vers la version 18.2.0 et réinstaller toutes les dépendances. Cette expérience m'a appris à gérer les conflits de versions, vérifier les peer dependencies et tester le build localement avant le déploiement.

Pièges à éviter

  • Modifier l'état directement au lieu d'utiliser la fonction setState
  • Ne pas passer correctement les callbacks aux composants enfants
  • Oublier la clé unique (key) pour chaque élément dans un map
  • Ignorer les conflits de versions entre React et ses dépendances, ce qui peut bloquer le build ou le déploiement
