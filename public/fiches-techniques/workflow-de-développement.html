Workflow de Développement

Définition concise

Le workflow de développement correspond à l’ensemble des étapes suivies pour concevoir, structurer, développer, tester et améliorer un projet web. Il permet d’assurer une progression logique, d’éviter les erreurs répétitives et de maintenir une cohérence globale.

Workflow appliqué dans mon projet Portfolio2025

1. Analyse et planification

Avant d’écrire la première ligne de code, j’ai défini les sections principales du site (accueil, projets, fiches techniques, réflexions) ainsi que la logique de navigation. Cette étape m’a permis d’anticiper l’organisation des dossiers et la structure globale.

2. Mise en place de la structure

J’ai ensuite créé l’architecture des dossiers (public, assets, projets, fiches, etc.) et configuré un serveur Node.js avec Express pour servir les fichiers statiques. Cette base technique m’a offert un environnement stable pour développer progressivement.

3. Développement progressif

J’ai travaillé page par page en respectant la séparation des responsabilités :

  • Structure en HTML avec balises sémantiques
  • Mise en page responsive avec Flexbox et CSS Grid
  • Ajout d’interactions dynamiques en JavaScript

4. Tests et ajustements

Après chaque ajout important, j’ai testé le site sur différentes tailles d’écran et utilisé les DevTools du navigateur pour corriger les problèmes d’alignement, d’espacement ou de responsive. Plusieurs ajustements ont été nécessaires après visualisation réelle.

5. Refactorisation

Après l’ajout de nouvelles pages, j’ai restructuré certains dossiers et harmonisé les noms de classes CSS afin d’améliorer la lisibilité et la cohérence du projet. Cette phase m’a permis de comprendre l’importance d’une maintenance continue.

Outils utilisés

  • Visual Studio Code pour le développement
  • Git et GitHub pour le versioning et le suivi des versions
  • Node.js et Express pour le serveur local
  • DevTools pour le debug et les tests responsive

Erreurs rencontrées

  • Commencer certaines sections sans plan détaillé
  • Modifier rapidement le CSS sans tester immédiatement sur mobile
  • Reporter la refactorisation à trop tard

Leçons apprises

Ce projet m’a appris qu’un workflow structuré réduit le stress et les corrections tardives. Planifier la structure en amont, tester régulièrement et refactoriser progressivement améliore considérablement la qualité et la maintenabilité du code.
